# 修繕積立金シミュレーター（すまい・る債運用対応）

マンション管理組合の **修繕積立金キャッシュフロー** を長期修繕計画にもとづいて試算し、
**すまい・る債で運用した場合のメリット** と **将来の資金不足（資金ショート）** を可視化する
ブラウザアプリです。バックエンド不要・全てブラウザ内で計算します。

## 使い方

```bash
npm install
npm run dev      # 開発サーバ（http://localhost:5173）
npm run build    # 本番ビルド（dist/ を静的配信。理事会への共有も可）
npm test         # 計算エンジンのテスト
```

## デプロイ（GitHub Pages）

シミュレーション・グラフはすべてブラウザ内で完結する静的SPAなので GitHub Pages で公開できます。
`.github/workflows/deploy.yml`（同梱済み）が push で自動ビルド＆デプロイします。

初回セットアップ:

```bash
git init
git add -A
git commit -m "initial: 修繕積立金シミュレーター"
git branch -M main
gh repo create <リポジトリ名> --public --source=. --remote=origin --push
#   ↑ gh CLI が無ければ、GitHubでリポジトリを作成後:
#   git remote add origin https://github.com/<user>/<repo>.git && git push -u origin main
```

その後、GitHub の **Settings → Pages → Build and deployment → Source を「GitHub Actions」** に設定。
以降 `main` に push するたびに `https://<user>.github.io/<repo>/` へ自動公開されます
（`vite.config.ts` の `base: './'` によりサブパス配信でもアセットが解決されます）。

> **注意：📄 PDFレポート出力はローカル（`npm run dev`）専用**です。
> レポート生成は Node＋ヘッドレスChrome（＋任意で `claude -p`）を使う `/api/report` に依存するため、
> 静的ホスティングでは動きません。公開版では自動的にボタンを隠し、代わりに案内文を表示します。
> レポートPDFが必要なときは手元で `npm run dev` → ボタン、または `npm run report` を実行してください。

## モデル化しているすまい・る債のルール

| 項目 | ルール |
|---|---|
| 購入単位 | 1口 50万円 |
| 購入回数 | 年1回・最大10回（10年）継続、毎年同一口数 |
| 購入上限 | その年の修繕積立金収入 ＋ 前年度末の積立金残高 の範囲内 |
| 利率 | 発行年度ごとに固定（「10年満期時 年平均利率」をフラット近似） |
| 利息 | 発行翌年から満期年まで毎年受取 |
| 償還 | 発行から10年後に元本償還 |
| 中途換金 | 発行から1年経過後に可能（資金不足時の充当に使用） |
| 認定優遇 | 管理計画認定マンションは利率を上乗せ |

> 利率は毎年度 住宅金融支援機構が決定します。本ツールは検討用の概算であり、
> 実際の利息・運用を保証するものではありません。
> 参考: 2025年度の10年満期時年平均利率は 0.525%（認定マンション 0.575%）。
> 2026年度から制度が拡充されているため、最新の条件・利率は公式サイトで確認してください。

## 実データの入れ方（CSV / JSON）

コードを触らず、ブラウザの **「CSV / JSON を読み込む」** から投入できます。
`samples/` に記入例があります（アプリの「記入用テンプレート」ボタンでも空CSVを取得可）。

| ファイル | ヘッダー（列） | 内容 |
|---|---|---|
| `reserve_steps.csv` | `適用開始年度, 戸あたり月額` | 段階増額計画 |
| `repair_plan.csv` | `年度, 金額, 項目` | 長期修繕計画の支出 |
| `other_cashflows.csv` | `年度, 金額, 摘要` | その他収支（正=収入 / 負=支出。管理費余剰の繰入もここ） |

- 種別は **ヘッダーの見出しで自動判定**（列の順序が多少違ってもOK）。
- 金額は `80000000` でも `8,000万` でも `¥8000万` でも解釈します。
- 複数ファイルをまとめて選択できます。
- 戸数・竣工年・開始残高・物価上昇率・認定有無は左パネルで設定し、
  **「現在の設定をJSONで書き出す」** で全体（CSV由来データ含む）を1ファイル保存・復元できます。

> 会計区分は現在「修繕積立金会計」のみをモデル化しています。
> 管理費会計からの繰入がある場合は `other_cashflows.csv` に収入として計上してください。

開発者向け: `src/engine/sampleData.ts` の初期値を直接書き換えることもできます。
